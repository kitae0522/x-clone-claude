# 계획 수립 워크플로

다음 작업에 대한 계획을 세워줘: $ARGUMENTS

## 필수 단계
1. `docs/TODO.md`를 읽고 현재 진행 상황을 파악
2. `docs/CONTEXT.md`를 읽고 이전 의사결정 확인
3. `docs/PLAN.md`를 읽고 전체 아키텍처 확인
4. 다음 형식으로 계획을 작성:

### 계획서 형식
- **목표**: 무엇을 달성할 것인지
- **수정/생성할 파일 목록** (백엔드/프론트엔드 구분)
- **구현 순서**: 번호 매긴 단계별 계획
- **인터페이스 정의**: 새로 만들 Go interface가 있다면 미리 정의
- **ReBAC 고려사항**: 접근 제어가 필요한 부분
- **위험 요소**: 주의해야 할 점

5. **계획서를 보여주고 반드시 멈춰라**
6. 내가 "승인" 또는 "ㅇㅇ"이라고 말하기 전까지 코드를 작성하지 마라
7. 승인 후: `docs/TODO.md`에 해당 작업을 추가하고 작업 시작
8. 작업 완료 후: `docs/TODO.md` 체크 표시, `docs/CONTEXT.md`에 결정사항 기록
